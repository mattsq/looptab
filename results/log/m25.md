# M25 — DONE (clean NEGATIVE for the loop thesis; verdict SHARPENED post-adversarial-review — the M20 result HOLDS for the cross-cell mixer too). The M23/M24 cross-cell MIXER re-test carried to the real multi-label datasets (`yeast`, `scene`, `emotions`) — the last open piece of the mixer arc (§9.4 bridge × the M24 mixer finding). **Under the honest metric (micro/macro-F1, 10-fold CV), the mixer does NOT beat a shallow joint MLP anywhere: it TIES `ff_matched` on the primary metrics (micro-F1, per-cell accuracy) on BOTH large datasets** — yeast micro-F1 Δ(mixer−ff) −0.005 (5/5, p=1.0), scene −0.005 (4/6, ns). The mixing OPERATOR adds nothing over the flat loop (Δ(mixer−flat) ns-to-neg), the mixing ARCHITECTURE adds nothing over the MLP (Δ(untied_mixer_matched−ff) ns on every metric both datasets), and the ONLY significant mixer deficit — the TIED mixer on yeast macro-F1/EM — is a **TYING** effect (not the padding artifact the review first caught, and not the mixing architecture): Δ(trm_mixer − untied_mixer_matched) macro-F1 −0.028 (1/9, p=.021). The one robust positive is leg-1 (joint > binary-relevance) on F1, which M20 already showed is joint-*modeling*, not a loop/mixer property (a plain MLP gets it).

**Why this test.** M20 gave the FLAT `trm` loop a fair shot on real multi-label tabular and it did NOT transfer (every loop-specific edge EM-only, tied `ff_matched` under F1). M23/M24 then showed the flat update CANNOT mix cells, and the cross-cell mixer (`trm_mixer`) SOLVES synthetic tasks the flat loop can't — but only where outputs are CROSS-CELL COUPLED (`converge` local, `hopfield` dense, `disruption` MED). Real multi-label labels are coupled by construction (yeast: gene-function co-occurrence, cardinality 4.24; scene: near mutual-exclusion, cardinality 1.07), so this is the honest test of whether mixing crosses over where the flat loop tied. Judged on **F1, not EM** (M20 showed the loop's EM "win" was a modal-label-combo artifact), with the M24e attribution controls so any win is honestly split architecture-vs-recurrence.

**★ ADVERSARIAL-REVIEW FIX (a BLOCKER-adjacent MAJOR that SHARPENED the negative).** The first yeast run (contiguous padding, canonical `..._20260705T120725`) appeared to show the mixer "SIGNIFICANTLY WORSE on yeast (macro-F1 −0.033, 0/10, p=.002)". An adversarial PR review caught the confound and it was VERIFIED: contiguous padding put ALL 9 zero columns into mixer **cell 13**, which predicts **label 13 — yeast's RAREST label (base rate 0.014)** — giving that one output cell ZERO direct input while `ff_matched` (flat vector) saw everything. That asymmetric per-cell handicap fell on macro-F1 (which weights rare labels equally) but not micro-F1/accuracy, and it appeared in BOTH mixer arms (tied AND untied) — the dead-cell signature. The original writeup's caveat "(ii) pad columns only dilute the mixer, can't explain its loss" was WRONG. **Fix:** distribute the pad one-per-cell (no dead cell; verified 0 dead cells, still 9 zero cols), and RE-RAN yeast (canonical `..._20260705T164426`). Result — the fix worked and cleaned up the picture: yeast micro-F1 Δ(mixer−ff) −0.012 → **−0.005 (p=1.0, a clean tie)**, and the mixing-ARCHITECTURE macro deficit VANISHED (Δ(untied_mixer_matched−ff) macro-F1 −0.013 [0/10, p=.002] → **−0.005 [4/6, p=.75, ns]**). The residual tied-mixer macro deficit PERSISTED (−0.032, p=.021), which the distributed rerun re-attributes cleanly to TYING (Δ(trm_mixer−untied_mixer_matched) macro −0.028, p=.021), not the artifact. Net: the review made the negative CLEANER (mixer ties the MLP on the primary metrics) and more precisely attributed.

**Mechanistic why (the honest reason mixing doesn't cross over).** In the synthetic mixer WINS the INPUT cells correspond positionally to the OUTPUT cells (same ring/grid/graph node), so token-mixing propagates constraints between structurally-aligned input↔output cells — the cross-cell dependency lives in a shared cell topology. In real multi-label the reshape `X → (n_cells=L, cell_dim=d/L)` groups ARBITRARY contiguous feature-blocks into label-cells with NO input↔output correspondence (feature block i has nothing to do with label i), so the token-mix has no meaningful structure to exploit — it is just a differently-parameterized MLP, and a plain joint MLP does at least as well. So M24's "cross-cell dependency" requirement is sharper than "the outputs are correlated": it needs coupling in a **shared input/output cell topology**, which real tabular's feature→label map lacks. **This is a scope limit, not proof mixing can't help real multi-label:** the naive reshape handicaps the mixer by construction (an adversarial-review framing point), so M25 establishes "the NAIVE reshape doesn't transfer"; a learned input→cell projection (or a feature-grouping aligned to label structure) is the untried steelman that would make the test falsifiable rather than foregone — left for future work.

**Verdict.** The cross-cell mixer does NOT transfer to real multi-label tabular. Under micro/macro-F1 + 10-fold CV, `trm_mixer` TIES a shallow joint MLP on the primary metrics on both large datasets (yeast a clean p=1.0 tie after the dead-cell fix; scene ns), and its only significant effect is a tied-mixer rare-label macro-F1/EM DEFICIT on yeast (a TYING cost, p=.021). Neither the mixing operator (vs flat loop), the mixing architecture (vs MLP), nor tying buys anything positive; the flat loop itself ties ff (M20 replication, sign-tested). The synthetic mixer wins (M23/M24) require a shared input/output cell topology that real multi-label label-coupling lacks — so the M20 negative STANDS, now confirmed for the mixer architecture as well as the flat loop. The only robust real-tabular positive remains leg-1 (joint output modeling > binary-relevance), not a loop/mixer property. This closes the mixer arc.
